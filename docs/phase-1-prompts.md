## 提示词 #1：项目启动

> 帮我从零设计一款"AI 圆桌讨论"Web 应用。项目构建按照多范式融合的工程化架构(SDD+DDD+TDD)
>
> - SDD(Spec/Schema-Driven Development,契约/模型驱动):前期基于业务需求，驱动数据建模(讨论、嘉宾、发言、共识/分歧等实体)与API契约的精确定义。
> - DDD(Design-Driven Development, 设计驱动):中期必须使用 UI UX Pro Max，以演播厅视觉风格与交互设计要素深度驱动前端组件与状态流转。
> - TDD(Test-Driven Development,测试驱动):后期针对核心逻辑(嘉宾生成、发言调度、共识提炼)严格遵循测试驱动流程，并包含完整的端到端(E2E)测试。
>
> ## 产品定义
>
> "AI Panel Studio"——用户输入任意讨论话题并指定专家人数(默认4人)，系统调用大模型动态生成主持人与专家嘉宾（姓名、职业/Title、立场、专属颜色），用户确认阵容后进入演播厅，观看一场 AI 驱动、实时推进的圆桌讨论。
>
> ## 核心功能
>
> - 首页：进行中讨论列表，支持加入已有讨论或发起新讨论
> - 嘉宾生成：用户输入话题+专家人数，LLM 生成主持人+专家阵容，用户可编辑确认
> - 演播厅：
>   - 主持人：开场、提问、串场、总结
>   - 专家：根据当前 transcript 自主决定发言（举手/抢答/补充/反驳），每次 1-2 句，禁止机械轮替
>   - 专家状态小窗：实时显示状态（思考中/发言中等），不展示 chain-of-thought
>   - 共识与分歧：独立观察员实时提炼，独立区域持续更新，不等讨论结束
>   - Transcript：发言人姓名+Title+颜色标识，屏蔽"举手"等内部动作
>   - 总结：主持人自然语言总结，页面禁止出现 JSON 或格式化字符
>
> ## 技术约束
>
> - 前后端分离。前端 Vite + React + TypeScript，后端 FastAPI + uv
> - LLM: DeepSeek API (BASE_URL=https://api.deepseek.com/v1, MODEL=deepseek-chat)
> - API Key 仅在后端环境变量读取
> - 数据库: SQLite
> - 实时通信: WebSocket
> - 多讨论并行隔离（状态、事件流、transcript、共识/分歧）
