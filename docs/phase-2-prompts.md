## 提示词 #1：进入 SDD 阶段

> 你现在处于 SDD（System Design & Data Modeling）阶段。
>
> 此前已经完成了完整的需求文档，请严格依据需求文档开展设计工作。
>
> 当前阶段目标：
>
> 将需求文档转化为系统设计方案，为后续 DDD、TDD、E2E 阶段提供基础。
>
> 注意：
>
> * 不编写任何业务代码
> * 不生成前端页面代码
> * 不生成测试代码
> * 不进入DDD、TDD、E2E阶段
> * 重点关注系统结构、数据模型、接口契约和开发规范
>
> 请完成以下设计文档：
>
> 1. API_CONTRACT.md
> * 梳理系统所有核心API
> * 明确请求参数、返回参数和错误码规范
> * 定义WebSocket事件协议
>
> 2. DATABASE_DESIGN.md
> * 完成数据库实体设计
> * 明确表结构、字段和关系
> * 给出核心索引设计
>
> 3. ER_DIAGRAM.md
> * 根据数据库设计生成ER图
> * 展示实体关系
>
> 4. APP_FLOW.md
> * 描述用户使用流程
> * 描述页面跳转关系
> * 描述核心交互逻辑
> * 描述系统事件流
>
> 5. TECH_STACK.md
> * 确定前端、后端、数据库、部署方案
> * 明确技术选型及版本
> * 说明选型理由
>
> 6. FRONTEND_GUIDELINES.md
> * 定义前端目录结构
> * 定义路由规范
> * 定义组件规范
> * 定义状态管理规范
> * 定义WebSocket使用规范
>
> 7. BACKEND_STRUCTURE.md
> * 定义后端目录结构
> * 定义模块划分
> * 定义表结构汇总
> * 定义API模块划分
> * 定义Agent编排模块结构
>
> 8. IMPLEMENTATION_PLAN.md
> * 制定项目开发计划
> * 划分开发阶段
> * 明确各阶段目标、交付物和验收标准
> * 为后续TDD阶段做好准备
>
> 输出要求：
>
> * 以Markdown文档形式组织结果
> * 每个文档关注设计内容即可
> * 不需要生成过于详细的实现细节
> * 优先保证架构合理性与工程可落地性
> * 所有设计必须能够支撑后续DDD、TDD、E2E阶段开发
>
> 当前仅输出SDD阶段成果。
