# AI Panel Studio — 应用流程文档

> **阶段**: SDD  
> **日期**: 2026-06-17

---

## 1. 页面路由规划

```
/                     → 首页（讨论列表）
/create               → 创建讨论（话题+专家人数）
/create/:id/panel     → 嘉宾阵容生成与编辑
/studio/:id           → 演播厅（实时讨论）
/report/:id           → 讨论总结报告
```

---

## 2. 核心用户流程

### 2.1 流程一：发起新讨论

```
┌──────────┐    ┌──────────────┐    ┌──────────────────┐
│   首页    │───▶│  创建讨论     │───▶│  嘉宾阵容生成     │
│  /       │    │  /create     │    │  /create/:id/panel│
└──────────┘    └──────────────┘    └────────┬─────────┘
                                             │
                                    ┌────────┴─────────┐
                                    │  用户编辑嘉宾?     │
                                    │  - 修改姓名/Title  │
                                    │  - 调整立场/颜色   │
                                    │  - 单体重生        │
                                    │  - 全体重生        │
                                    └────────┬─────────┘
                                             │
                                    ┌────────┴─────────┐
                                    │  确认阵容          │
                                    │  PUT /panel       │
                                    └────────┬─────────┘
                                             │
                                    ┌────────┴─────────┐
                                    │  进入演播厅        │
                                    │  /studio/:id      │
                                    └──────────────────┘
```

**步骤细节**:

1. **首页 → 创建讨论**: 用户点击"发起新讨论"按钮
2. **输入话题**: 文本框，建议最长 200 字，实时字符计数
3. **选择专家人数**: 滑块或数字输入，范围 2-8，默认 4
4. **调用 LLM 生成**: 点击"生成嘉宾阵容"，前端显示加载状态（POST /panel/generate）
5. **编辑确认**: 生成完成后展示嘉宾卡片，用户可逐人编辑或重生成
6. **确认进入**: 确认后调用 PUT /panel，跳转演播厅

### 2.2 流程二：加入已有讨论并观看

```
┌──────────┐    ┌──────────────────┐    ┌──────────────────┐
│   首页    │───▶│  演播厅（观看）    │───▶│  讨论报告         │
│  /       │    │  /studio/:id     │    │  /report/:id     │
└──────────┘    └──────────────────┘    └──────────────────┘
```

**步骤细节**:

1. 首页展示"进行中"和"已结束"两个 Tab
2. 点击"进行中"讨论卡 → 进入演播厅实时观看
3. 点击"已结束"讨论卡 → 进入讨论报告页面
4. 讨论结束后，演播厅内自动出现"查看报告"入口

## 4. 核心交互逻辑

### 4.1 创建讨论页交互

| 状态 | 页面表现 |
|------|----------|
| 话题为空 | "生成嘉宾阵容"按钮禁用，提示"请输入讨论话题" |
| 话题过长 | 输入框变红，计数器显示超限字数 |
| 生成中 | 按钮显示加载动画，页面不可操作 |
| 生成失败 | Toast 错误提示，可重试 |
| 生成成功 | 自动跳转到嘉宾编辑页 |

### 4.2 嘉宾编辑页交互

| 操作 | 交互 |
|------|------|
| 编辑嘉宾 | 点击嘉宾卡片 → 弹出编辑面板（姓名/Title/立场/颜色拾取器） |
| 颜色编辑 | 预设颜色板 + 自定义 HEX 输入 |
| 单体重生 | 嘉宾卡片的"重新生成"按钮 → 仅替换该位 |
| 全体重生 | 顶部"全部重新生成"按钮 → 二次确认弹窗 |
| 确认阵容 | 底部"确认阵容进入演播厅"按钮 → 确认后跳转 |

### 4.3 演播厅页交互

| 区域 | 行为 |
|------|------|
| 专家状态栏（左/顶） | 实时显示每位专家状态灯（绿=idle，黄=preparing，蓝=speaking），下方展示关注点摘要 |
| Transcript（中） | 流式逐字显示发言，发言人名称+颜色标记，滚动自动跟底 |
| 共识/分歧（右） | 列表项动态插入，动画过渡，绿色共识/橙色分歧标签 |
| 控制栏 | 暂停/继续/下一轮/结束 四个按钮，暂停时出现"继续"按钮 |
| 主持人总结 | 讨论结束时，在 Transcript 区域以特殊样式（带边框高亮）显示 |

## 5. 系统事件流

### 5.1 一轮发言的完整事件序列

```
时间轴 →
─────────────────────────────────────────────────────────────▶

1. expert_status: 所有专家 → idle (等待中)
2. expert_status: 专家A → preparing (开始思考), desire_value=0.72
3. expert_status: 专家B → preparing, desire_value=0.85
4. expert_status: 主持人 → preparing, desire_value=0.81
5. [Scheduler 选择 desire 最高者: 专家B]
6. expert_status: 专家B → speaking, focus_summary="正在阐述..."
7. expert_status: 专家A → idle (放弃本轮)
8. expert_status: 主持人 → idle
9. utterance_token: 专家B 逐 token 推送发言...
10. utterance_token: 专家B 最后 token (is_last=true)
11. utterance_complete: 专家B 发言完成
12. [观察员分析中]
13. consensus_update: 发现新共识/或未发现
14. expert_status: 专家B → idle
─────────────────────────────────────────────────────────────▶
     (如果未结束，等待下一轮触发，重复 1-14)
```

### 5.2 结束流程事件序列

```
触发条件: 用户点结束 / max_rounds 达到 / rounds_without_consensus >= threshold
─────────────────────────────────────────────────────────────▶
1. discussion_control: max_rounds_reached (如果是自动触发)
2. [主持人获得发言权]
3. expert_status: 主持人 → preparing
4. expert_status: 主持人 → speaking
5. utterance_token: 主持人总结 (last utterance)
6. utterance_complete: 主持人总结完成 (type=summary)
7. discussion_ended: end_reason=...
─────────────────────────────────────────────────────────────▶
```
